# BZip2 Compression Pipeline Implementation

## Project Overview
This project is a complete, from-scratch implementation of the BZip2 data compression algorithm, written in C. It was developed as a comprehensive project to demonstrate advanced data structures, algorithmic efficiency, and low-level memory management.

The pipeline successfully implements both encoding and decoding pathways, ensuring lossless data compression verified by byte-for-byte round-trip testing.

## Implementation Details

### Stage 1: BWT & RLE-1 (Enhanced)
* **Block Division:** Dynamically parses large files into configurable memory blocks (100KB - 900KB) defined via `config.ini`.
* **Enhanced RLE-1:** Implemented an "Extra Marks" enhancement by utilizing a dynamic Run-Length Threshold. Instead of hardcoding a minimum run of 4, the threshold is loaded from configuration, preventing negative compression on highly varied data.
* **Suffix-Array BWT:** Implemented the O(N log^2 N) Prefix-Doubling Suffix Array algorithm for the Burrows-Wheeler Transform, bypassing the severely inefficient O(N^2) matrix sorting approach for massive performance gains.

### Stage 2: Move-to-Front (MTF) & RLE-2
* **MTF Transform:** Utilizes a fast O(N) local-shifting palette (0-255) to convert repeating byte patterns from the BWT output into localized low-number indices, heavily skewing the data toward zero.
* **RLE-2 (Zero-Run Encoding):** A highly specialized Run-Length Encoder designed exclusively to target the massive influx of zeros generated by the MTF stage, drastically shrinking the data footprint before entropy coding.

### Stage 3: Canonical Huffman Coding
* **Depth-Limited Frequency Scaling:** Engineered a dynamic frequency scaler to prevent 16-bit integer overflows when the MTF stage produces highly unbalanced zero-heavy distributions, guaranteeing the tree depth never exceeds safe memory bounds.
* **Canonical Tree Generation:** Generates optimized bit-lengths rather than a full binary tree structure, allowing the decoder to mathematically reconstruct the exact tree utilizing only a tiny 256-byte header.
* **Bit-Level Packing:** Packs variable-length Huffman codes into raw binary bytes using custom bit-shifting stream buffers for maximum compression density.

## Build Instructions

### For Linux (Automated via Makefile)
The project includes a cross-platform Makefile for seamless compilation.
1. Open your terminal in the project directory.
2. Run `make all` to compile the native executable.
3. Run `./bzip2_impl` to execute.
*(Note: To cross-compile a Windows executable from Linux, run `make windows`).*

### For Windows (Visual Studio)
1. Create a new Empty Project in Visual Studio.
2. Add all `.c` files from the `src/` folder to "Source Files".
3. Add `project.h` from `include/` to "Header Files".
4. Ensure `config.ini` and `input.txt` are in the working directory.
5. Press F5 to build and run.

## Usage Examples

1. Place the file you wish to compress in the main directory and rename it to `input.txt`.
2. Ensure your `config.ini` is formatted correctly in the same directory.
3. Run the executable. 

**Automated Pipeline Actions:**
* The program will read the configuration and execute the pipeline.
* It will generate the compressed binary file (`compressed`).
* It will instantly decode the binary to `restored_output.txt` and perform a byte-for-byte memory comparison to verify 100% data integrity.
* It will append the performance metrics to `results/results.csv`.

**Graph Generation:**
To visualize the results, ensure you have Python 3 installed and run:
`python3 generate_graphs.py`
This will parse the CSV and output a performance bar chart to the `results/` folder.
